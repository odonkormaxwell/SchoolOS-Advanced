# SMS COMMUNICATION MODULE

## Objective

Build a centralized SMS communication system for Edulex that allows schools to send branded SMS messages to parents, guardians, students, and staff without requiring each school to create its own SMS provider account.

Edulex should manage the SMS infrastructure centrally while allowing each school to have its own approved Sender ID.

The goal is to make SMS setup simple for schools while preserving their brand identity.

---

# SMS ARCHITECTURE

Edulex should own and manage the SMS provider account.

Recommended Provider:

Arkesel SMS API

Future Providers:

* Hubtel
* mNotify
* Nalo Solutions

Schools should NOT be required to:

* Create Arkesel accounts
* Manage API keys
* Configure SMS gateways
* Purchase SMS directly from providers

Edulex should handle all SMS infrastructure.

---

# SMS FLOW

School Admin sends SMS

↓

Edulex SMS Gateway

↓

Arkesel API

↓

Parent / Student / Staff

SMS should appear from the school's approved Sender ID.

Example:

From: VenusBaby

Dear Parent,
Michael Ofori was absent from school today.

Not:

From: EDULEX

# SENDER ID MANAGEMENT

Each school should have its own Sender ID.

The Sender ID should be displayed exactly as approved.

Do NOT force uppercase.

Allow normal branding.

Examples:

* Stanbic
* Fidelity
* VenusBaby
* StMarys
* RoyalInt
* BlueCrest
* Edulex

Do not automatically convert to:

* STANBIC
* FIDELITY
* VENUSBABY

---

# SENDER ID RULES

Allow:

* Uppercase letters
* Lowercase letters
* Numbers

Examples:

* Stanbic
* VBNest
* Edulex
* ABC123

Disallow:

* Special characters
* Multiple spaces
* Emojis

Examples:

Invalid:

* St. Mary's
* Royal@School
* School#
* Venus Baby Nest

---

# CHARACTER LIMIT

Maximum Sender ID Length:

11 Characters

Display live character counter.

Example:

Sender ID

VenusBaby

9 / 11 Characters

# REQUEST SENDER ID

Create button:

Request Sender ID

Workflow:

1. School enters Sender ID.
2. System validates length.
3. Request submitted.
4. Super Admin reviews.
5. Sender ID approved or rejected.
6. Status updated.

Statuses:

* Pending
* Approved
* Rejected
* Suspended
